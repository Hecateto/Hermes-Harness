长对话记忆丢失问题
问题描述
用户存在倾向于复用同一个session的情况。对话超出最大上下文导致对前文细粒度信息的丢失。
例如，在一次涉及检索的workflow中，agent调用SQL操作检索得到了一个URL，后续间隔多轮对话，用户提及该事件和URL，遗忘该细粒度网址。
抽象为：有限上下文与无限交互意图之间的工程博弈

问题分解
  
- 模型：上下文窗口限制， 注意力稀释和衰减
  
- Tool返回的非结构化文本信息密度不均
  
- Session管理策略设计
  
- 用户使用习惯
  
LLM给出的一些Insight
  
- 设计一个 Pin Slot 用于关键Entity信息的固定，本质上也可以视作一种记忆持久化的组件；相应地，需要设计对Tool Result的记忆管理策略，这里可以参考CC, Hermes等优质Harness的design；

- 设计Session的生命周期管理和自动更新；本质上还是Session切换，存在记忆压缩和丢失问题；

- 一些Entity重要性评估的规则：（ Prompt / Harness 规定约束； 设计Memory机制，比如冷热度指标等）
- 信息熵与生成难度，保留高熵硬事实（推理难以生成的）
- 核心交付物，保留结果抛弃过程
- 针对用户效用反馈进行强记忆

从理论视角出发
- 首先，无损压缩是理论不可能的
- 一些转变：不要求 C(S) 包含所有信息，而要求 C(S) 包含"足够找到信息的索引"，这里对应CC的检索指针设计
